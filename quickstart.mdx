<Warning>
  Only use this tool against networks and systems you own or have explicit written authorization to test. Unauthorized access to computer networks is illegal in most jurisdictions. The authors accept no responsibility for misuse.
</Warning>

<Note>
  You need a wireless adapter that supports **monitor mode** and **packet injection**. Common compatible chipsets include Atheros AR9271 and Realtek RTL8187. Built-in laptop adapters typically do not support monitor mode. If you are unsure, check your adapter with `iw list` and look for `* monitor` under supported interface modes.
</Note>

<Steps>

  <Step title="Install wireless tools (first run only)">
    Before using the wireless features, install the required tools by entering **5**:

    ```text
    Enter your choice: 5
    ```

    The tool installs: `aircrack-ng`, `crunch`, `xterm`, `wordlists`, `reaver`, `pixiewps`, `bully`, `wifite`, `airodump-ng`, `nmap`, `metasploit-framework`, `hydra`, `wifiphisher`, `nikto`, `netcat`, `gobuster`, `ncat`, and `sqlmap`.

    ```text
    Installing Necessary Tools...
    Installing aircrack-ng...
    Installing crunch...
    Installing reaver...
    ...
    ```

    <Tip>
      On Kali Linux most of these packages come pre-installed. Skip this step if you already have them — running it again on an existing install is safe and will not overwrite existing configurations.
    </Tip>
  </Step>

  <Step title="Start monitor mode">
    Re-open the menu (or stay in the same session) and enter **1**:

    ```text
    Enter your choice: 1
    Enter the interface: wlan0
    ```

    The tool runs `airmon-ng start wlan0 && airmon-ng check kill`, which:

    1. Switches `wlan0` to monitor mode, typically renaming it to `wlan0mon`.
    2. Kills background processes (NetworkManager, wpa_supplicant) that would interfere with packet capture.

    ```text
    Starting monitor mode on wlan0...

    Found 2 processes that could cause trouble.
    Kill them using 'airmon-ng check kill' before putting
    the card in monitor mode, they will interfere by changing channels
    and sometimes putting the interface back in managed mode

      PID Name
     1234 NetworkManager
     5678 wpa_supplicant

    PHY     Interface       Driver          Chipset
    phy0    wlan0           ath9k_htc       Atheros AR9271

                    (mac80211 monitor mode vif enabled for [phy0]wlan0 on [phy0]wlan0mon)
                    (mac80211 station mode vif disabled for [phy0]wlan0)
    ```

    Your adapter is now in monitor mode as `wlan0mon`.
  </Step>

  <Step title="Scan for nearby networks">
    Enter **3** from the menu:

    ```text
    Enter your choice: 3
    Enter the interface: wlan0mon
    ```

    The tool runs `airodump-ng wlan0mon -M`, opening a live capture window that lists all nearby access points:

    ```text
     CH  9 ][ Elapsed: 12 s ][ 2024-03-15 14:22

     BSSID              PWR  Beacons    #Data, #/s  CH   MB   ENC CIPHER  AUTH ESSID

     AA:BB:CC:DD:EE:FF  -42       14        0    0   6  130   WPA2 CCMP   PSK  HomeNetwork
     11:22:33:44:55:66  -67        8        2    0  11   54   WPA2 CCMP   PSK  OfficeWiFi
     FF:EE:DD:CC:BB:AA  -81        5        0    0   1  130   WEP  WEP         OldRouter

     BSSID              STATION            PWR   Rate    Lost    Frames  Notes  Probes
    ```

    The columns you need for further testing:

    | Column | Description |
    |--------|-------------|
    | `BSSID` | MAC address of the access point |
    | `CH` | Wi-Fi channel the AP is broadcasting on |
    | `ENC` | Encryption type (WPA2, WPA, WEP, OPN) |
    | `CIPHER` | Cipher suite (CCMP, TKIP) |
    | `ESSID` | Human-readable network name |
    | `PWR` | Signal strength — less negative means stronger |

    Press `Ctrl+C` to stop the scan and return to the menu.
  </Step>

  <Step title="Stop monitor mode and restore networking">
    When you are done, restore your adapter to managed mode by entering **2**:

    ```text
    Enter your choice: 2
    Enter the interface: wlan0mon
    ```

    The tool runs `airmon-ng stop wlan0mon && service network-manager restart`, which:

    1. Disables monitor mode and renames the interface back to `wlan0`.
    2. Restarts NetworkManager so normal Wi-Fi connectivity is restored.

    ```text
    Stopping monitor mode on wlan0mon...

    PHY     Interface       Driver          Chipset
    phy0    wlan0mon        ath9k_htc       Atheros AR9271

                    (mac80211 station mode vif enabled on [phy0]wlan0)
                    (mac80211 monitor mode vif disabled for [phy0]wlan0mon)
    ```

    Your adapter is back in managed mode and NetworkManager will reconnect to saved networks automatically.
  </Step>
</Steps>
